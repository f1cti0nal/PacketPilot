# JA3S — server TLS fingerprint — design

Status: design · 2026-06-24 · Feature: add the **server**-side TLS fingerprint (`ja3s`) as a per-flow
column, completing the fingerprint matrix (client JA3/JA4 + server version/cipher already exist).

## Problem

PacketPilot fingerprints TLS *clients* (JA3/JA4) and records the negotiated server version/cipher, but
not the server's **JA3S** — `MD5("SSLVersion,Cipher,Extensions")` over the ServerHello. JA3S
identifies the server's TLS stack and, paired with the existing client-fingerprint malware matching,
extends C2 detection to the server side (specific C2 servers have known JA3S). The ServerHello parser
already exists (it reads version + cipher + iterates extensions for `supported_versions`); only the
fingerprint was missing.

## Approach

Engine-only, no new deps (reuses the vendored `fingerprint::md5_hex` + `is_grease`):
- `tls/mod.rs`: the ServerHello parser is refactored to return a `ServerHello { version, legacy_version,
  cipher, ext_types }` struct (it already walked the extensions — it now also collects the non-GREASE
  extension types in wire order). `ja3s_hash` = `MD5("{legacy_version},{cipher},{ext_types}")`.
  Crucially JA3S hashes the **legacy_version** (the wire ServerHello version, frozen `0x0303` for both
  TLS 1.2 and 1.3) — not the `supported_versions`-unmasked version used for the version/cipher columns
  and the weak-TLS check.
- `sniff_server_hello` now returns `(version_name, cipher_label, ja3s)`; `decode` sets `meta.ja3s`.
- The fingerprint threads through the per-flow-column pipeline like the other fingerprints:
  `PacketMeta.ja3s` → sticky `FlowRecord.ja3s` → Arrow/Parquet column 26 (`FLOW_PARQUET_VERSION` 7 → 8,
  29 columns) → DuckDB view → `ppcap-wasm` `FlowDto` → UI (JA3S chip + drawer field + search index).

## Scope

In: the JA3S server fingerprint single per-flow column. Out: matching JA3S against a server-side
malware-fingerprint feed (the feed is client JA3/JA4 today), cross-segment ServerHello reassembly
(shared with the existing path).

## Invariants

Engine-only; no new deps. The refactor preserves the existing `tls_version`/`tls_cipher` columns and
the weak-TLS check (both use the unmasked `version`). GREASE removed from the JA3S extensions (mirrors
JA3). Bounded + panic-free (unchanged parser bounds). Column shift to 29 with schema-drift / roundtrip
/ threat_e2e positional readers updated in lockstep.
