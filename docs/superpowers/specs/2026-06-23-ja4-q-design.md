# JA4-Q transport marker — Design Spec

**Status:** approved design, pre-plan
**Date:** 2026-06-23
**Branch:** `feat/ja4-q`

## Goal

Correct the JA4 transport marker for QUIC flows. QUIC SNI phase A (PR #27) feeds the recovered QUIC ClientHello to `fingerprint_tls_client_hello`, which hardcodes the protocol letter `t` (TCP) in the JA4 `a` segment (`fingerprint/mod.rs:293`). Per the FoxIO JA4 spec, QUIC fingerprints must use `q`. So today QUIC JA4s are mislabeled as TCP and won't match QUIC (`q`-prefixed) entries in the embedded malware fingerprint set. Only the leading protocol letter differs between the TCP and QUIC JA4 of the same ClientHello — everything else is computed identically.

## Architecture

Thread a small transport enum through the one fingerprint function. **JA3 is unchanged** — it carries no transport marker (verified: JA3 is `version,ciphers,extensions,curves,ecpf` MD5). Engine-only; no new deps; wasm-safe (no change to `ppcap-wasm`, which only passes the `ja4` string through).

**Tech stack:** Rust (ppcap-core). No new deps.

## Global Constraints

- **No regression to TCP JA4** — every existing `t…` JA4 (the golden test values + the embedded sig matching for TCP) must be byte-identical after this change. The TCP call sites pass the `Tcp` variant.
- **No new deps; wasm-safe; C-free** (no change to those invariants). Run cargo from `engine/`; `cargo fmt` before commit.
- **Minimal change** — only the JA4 `a` first character becomes parameterised; the JA4 `b`/`c` and the whole JA3 are untouched.

## Components

### 1. `engine/crates/ppcap-core/src/fingerprint/mod.rs`
```rust
/// Transport the ClientHello was carried over — sets the JA4 protocol letter (FoxIO spec).
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum Ja4Transport { Tcp, Quic }

impl Ja4Transport {
    fn marker(self) -> char {
        match self { Ja4Transport::Tcp => 't', Ja4Transport::Quic => 'q' }
    }
}
```
- `pub fn fingerprint_tls_client_hello(payload: &[u8], transport: Ja4Transport) -> Option<TlsFingerprints>`.
- Line 293 → `let ja4_a = format!("{}{ver_code}{sni_flag}{nc:02}{ne:02}{alpn}", transport.marker());`.
- Update the `:77` doc comment to describe the `t`/`q` protocol letter.

### 2. `engine/crates/ppcap-core/src/lib.rs`
Re-export `Ja4Transport` alongside `fingerprint_tls_client_hello`, `TlsFingerprints`.

### 3. `engine/crates/ppcap-core/src/decode/mod.rs`
- TCP branch (`:416`): `fingerprint_tls_client_hello(payload, Ja4Transport::Tcp)`.
- QUIC branch (`:453`): `fingerprint_tls_client_hello(&record, Ja4Transport::Quic)`.

### 4. Existing call sites (no behavior change)
The fingerprint unit tests (`:551 :575 :608 :634`) and the `tests/fingerprint.rs` integration calls (`:170 :290`) pass `Ja4Transport::Tcp` — preserving every current `t…` golden value.

## Data flow & error handling

A QUIC flow's ClientHello → decode QUIC branch → `fingerprint_tls_client_hello(ch, Quic)` → JA4 `q…` → stored on the flow → aggregated into `IpThreat.fingerprints` → matched against the embedded sig set with the correct `q` prefix. No new failure modes (the enum is infallible; the function's `Option` contract is unchanged).

## Testing

- **`fingerprint/mod.rs`:** a new test asserting that for the same ClientHello bytes, `fingerprint_tls_client_hello(ch, Ja4Transport::Quic).ja4` is byte-identical to the `Tcp` JA4 **except** the leading char is `q` (and the `Tcp` call still produces the existing golden `t…` value — unchanged). JA3 identical across both transports.
- **`decode`:** extend the phase-A QUIC integration test to assert the recovered JA4 starts with `q` (and the TCP-TLS decode test still yields a `t…` JA4).
- **Gate:** `cargo test -p ppcap-core` green; clippy `-D warnings` clean; C-free gate empty; the separate `ppcap-wasm` builds. Lockfiles unchanged (no new deps).

## Out of scope

- The DTLS `d` marker (no DTLS path exists). The larger JA4-QUIC variant that fingerprints QUIC **transport parameters** (a separate fingerprint, not the JA4 `a` letter). Any UI change — the `q` JA4 flows through the existing per-flow / `IpThreat.fingerprints` surfaces automatically. WASM-export/Tauri/CLI unchanged.
