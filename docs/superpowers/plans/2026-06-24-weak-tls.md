# Weak / deprecated TLS detector — implementation plan

Spec: [2026-06-24-weak-tls-design.md](../specs/2026-06-24-weak-tls-design.md)

One vertical PR (engine + UI), reusing the cert-health reassembler.

## Engine (`engine/crates/ppcap-core/`)

1. `tls/mod.rs`:
   - `WeakTlsReason` (`DeprecatedVersion { version }`, `WeakCipher { cipher, name, rank }`) with
     `kind_str` / `severity_rank` / `evidence`.
   - `parse_server_hello` + `parse_server_hello_body` (legacy_version, session_id skip, cipher,
     `supported_versions` 0x002b override), `weak_tls_reasons`, `WEAK_CIPHERS` table, `tls_version_name`.
   - `FlightBuf { bytes, weak_parsed }` replaces the bare `Vec<u8>` buffer value; `feed_server`
     retries `parse_server_hello` on the reassembled `buf.bytes` until the ServerHello is complete,
     recording one `WeakTlsObservation` per flight.
   - `into_results() -> ReassemblerResults { certs, weak_tls }` (replaces `into_observations`).
2. `model/finding.rs`: `FindingKind::WeakTls` + `as_str` `"weak_tls"`.
3. `detect/mod.rs`: tracker `weak_tls` map + `observe_weak_tls` + `WeakTlsCandidate` +
   `weak_tls_candidates`; `WeakTlsParams` + `detect_weak_tls` (severity by worst reason, T1040);
   `stage_ordinal` (3 / Collection), `stage_label` ("Collection"), `kind_phrase`.
4. `analyze/mod.rs`: `PipelineConfig.weak_tls`; fold `into_results().weak_tls` into the tracker;
   `findings.extend(detect_weak_tls(...))`.
5. `report/mod.rs`: `kind_label` arm. `lib.rs`: export `WeakTlsParams`.
6. Tests: ServerHello parse (deprecated version, weak cipher, SSLv3+NULL = High, strong = none,
   `supported_versions` unmasks TLS 1.3), split-across-segments regression, `detect_weak_tls` severity.

## UI (`ui/src/`)

7. `types.ts`: `"weak_tls"` in the `FindingKind` union.
8. `IncidentsPanel` + `IncidentHero` `KIND_META` (label "Weak TLS", `ShieldOff`); IncidentHero
   `KIND_STAGE` ("Collection").
9. Broaden `CertHealthPanel` to filter `tls_cert_health || weak_tls`, header "TLS issues"; update its test.

## Gates

Engine: `fmt` · `clippy -D warnings` · `test` · C-free gate · `wasm32`. UI: `build:wasm` ·
`test:coverage` (80/70) · `build`. Then adversarial review, PR, watch CI, merge on local gates.
