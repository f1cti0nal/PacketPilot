# ARP-spoofing detection — implementation plan

Spec: [2026-06-24-arp-spoof-design.md](../specs/2026-06-24-arp-spoof-design.md)

Engine-only L2 detector + UI finding-kind wiring. One PR.

## Engine (`engine/crates/ppcap-core/src/`)

1. `model/packet.rs`: `ArpClaim { sender_ip: Ipv4Addr, sender_mac: [u8;6] }` + `PacketMeta.arp`.
2. `model/finding.rs`: `FindingKind::ArpSpoof` + `as_str` → `"arp_spoof"`.
3. `decode/mod.rs`: `parse_arp(l3)` (IPv4-over-Ethernet; skip `0.0.0.0`) set on `meta.arp` in the ARP
   branch.
4. `detect/mod.rs`: `BehaviorTracker.arp: HashMap<IpAddr, HashSet<[u8;6]>>` (+ `new()` + bound
   `MAX_ARP_MACS`); `observe_arp`; `ArpSpoofCandidate` + `arp_spoof_candidates`; `ArpSpoofParams
   { enabled, min_macs: 2, ignore_ips }` + `detect_arp_spoof` (High, T1557.002); incident arms
   (Collection).
5. `analyze/mod.rs`: per-packet `observe_arp` when `meta.arp` is set; `PipelineConfig.arp_spoof` +
   `detect_arp_spoof`. `lib.rs`: re-export `ArpSpoofParams`. `report/mod.rs`: `kind_label`.

## UI (`ui/src/`)

6. `types.ts`: `FindingKind` += `"arp_spoof"`. `cockpit/IncidentHero.tsx` +
   `components/triage/IncidentsPanel.tsx`: `KIND_META` (Split), `KIND_STAGE` (Collection),
   `CONTACT_NOUN` ("MACs").

## Tests

7. `detect/mod.rs`: one IP / multiple MACs → flag; stable one-IP-one-MAC → none; `ignore_ips` exempts
   a migrating VIP. `decode/mod.rs`: `parse_arp` extracts the IPv4 claim; rejects short / `0.0.0.0` /
   non-Ethernet.

## Verify

Engine: full `cargo test -p ppcap-core` · `clippy`. UI: `test:coverage` · `build`. `build:wasm`.
Adversarial review focused on FP (VRRP/failover/DHCP) + parse correctness. Then PR, merge on local
gates.
