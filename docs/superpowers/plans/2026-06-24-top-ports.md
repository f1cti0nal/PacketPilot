# Top ports / services panel — implementation plan

Spec: [2026-06-24-top-ports-design.md](../specs/2026-06-24-top-ports-design.md)

Pure-UI, one PR. No engine change (`summary.port_histogram` already populated).

## UI (`ui/src/`)

1. `lib/services.ts`: `serviceName(port) -> string | null` from a well-known-port table.
2. `cockpit/TopPortsCard.tsx`: top 8 of `port_histogram` (sorted pkts desc, then bytes), each a row —
   port + transport + service (or "non-standard") + pkt/byte counts + a proportional bar. Uses the
   `Card` primitive. Hidden when empty.
3. `components/Dashboard.tsx`: import + render `<TopPortsCard ports={s.port_histogram ?? []} />` in
   the protocol/charts grid (next to the sunburst — balances the row).

## Tests

4. `lib/services.test.ts`: known ports named, unknown → null.
5. `cockpit/TopPortsCard.test.tsx`: renders the busiest ports with service names + the "non-standard"
   flag; hidden when empty.
