# Vertical port-scan detection — implementation plan

Spec: [2026-06-24-port-scan-design.md](../specs/2026-06-24-port-scan-design.md)

Engine-only behavioral detector + UI finding-kind wiring. One PR. Mirrors the host-sweep detector.

## Engine (`engine/crates/ppcap-core/src/`)

1. `model/finding.rs`: `FindingKind::PortScan` + `as_str` → `"port_scan"`.
2. `detect/mod.rs`:
   - `PortScanCandidate { src, dst, ports }` (mirrors `SweepCandidate`).
   - `BehaviorTracker.port_scan: HashMap<(IpAddr, IpAddr), HashSet<u16>>` (+ `new()`); fold in
     `observe_flow_contact` next to `fanout` (same bounds).
   - `port_scan_candidates(min_ports)`.
   - `PortScanParams { enabled, min_ports: 30, ignore_src }` + `detect_port_scan` (High, T1046,
     `dst_ip = Some` / `dst_port = None`).
   - incident-correlation arms: `stage_ordinal`(0 Discovery) / `stage_label` / `kind_phrase`.
3. `analyze/mod.rs`: `PipelineConfig.port_scan` + Default + `detect_port_scan`. `lib.rs`: re-export
   `PortScanParams`. `report/mod.rs`: `kind_label` → "Port Scan". (`export` Sigma category uses the
   `_ => firewall` catch-all — correct for a scan.)

## UI (`ui/src/`)

4. `types.ts`: `FindingKind` += `"port_scan"`.
5. `cockpit/IncidentHero.tsx` + `components/triage/IncidentsPanel.tsx`: `KIND_META` (ScanSearch),
   `KIND_STAGE` (Discovery), `CONTACT_NOUN` ("ports").

## Tests

6. `detect/mod.rs`: many-ports-on-one-host → flag; below-threshold + ports-spread-across-hosts (a
   sweep, not a scan) → none; `ignore_src` exempts a sanctioned scanner.

## Verify

Engine: full `cargo test -p ppcap-core` · `clippy`. UI: `test:coverage` · `build`. `build:wasm`.
Adversarial review focused on FP (FTP/RTP multi-port, aggregation). Then PR, merge on local gates.
