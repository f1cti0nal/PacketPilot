# SYN-flood / TCP-DoS detection — implementation plan

Spec: [2026-06-24-syn-flood-design.md](../specs/2026-06-24-syn-flood-design.md)

Engine-only behavioral detector + UI finding-kind wiring. One PR. Reuses the port-scan probe gate.

## Engine (`engine/crates/ppcap-core/src/`)

1. `model/finding.rs`: `FindingKind::SynFlood` + `as_str` → `"syn_flood"`.
2. `detect/mod.rs`:
   - `SynFloodStat { incomplete, sources }` + `BehaviorTracker.syn_flood: HashMap<(IpAddr, u16),
     SynFloodStat>` (+ `new()`).
   - fold inside the existing `!completed_session` branch of `observe_flow_contact` (per-`(dst, port)`
     incomplete count + bounded sources).
   - `SynFloodCandidate` + `syn_flood_candidates(min_incomplete)`.
   - `SynFloodParams { enabled, min_incomplete: 200, ignore_dst }` + `detect_syn_flood` (High,
     T1499.001; `dst_ip = None` / `dst_port = Some`); incident arms (Impact, ordinal 6).
3. `analyze/mod.rs`: `PipelineConfig.syn_flood` + Default + `detect_syn_flood`. `lib.rs`: re-export
   `SynFloodParams`. `report/mod.rs`: `kind_label` → "SYN Flood".

## UI (`ui/src/`)

4. `types.ts`: `FindingKind` += `"syn_flood"`. `cockpit/IncidentHero.tsx` +
   `components/triage/IncidentsPanel.tsx`: `KIND_META` (Waves), `KIND_STAGE` (Impact), `CONTACT_NOUN`
   ("half-open conns").

## Tests

5. `detect/mod.rs`: many half-open to one service → flag; completed sessions + below-threshold → none.
